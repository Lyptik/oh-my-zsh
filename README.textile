    <article class="markdown-body entry-content" itemprop="mainContentOfPage"><p><a href="https://camo.githubusercontent.com/5c385f15f3eaedb72cfcfbbaf75355b700ac0757/68747470733a2f2f73332e616d617a6f6e6177732e636f6d2f6f686d797a73682f6f682d6d792d7a73682d6c6f676f2e706e67" target="_blank"><img src="https://camo.githubusercontent.com/5c385f15f3eaedb72cfcfbbaf75355b700ac0757/68747470733a2f2f73332e616d617a6f6e6177732e636f6d2f6f686d797a73682f6f682d6d792d7a73682d6c6f676f2e706e67" alt="" data-canonical-src="https://s3.amazonaws.com/ohmyzsh/oh-my-zsh-logo.png" style="max-width:100%;"></a></p>
<p><code>oh-my-zsh</code> is an open source, community-driven framework for managing your <a href="http://www.zsh.org/">Zsh</a> configuration.</p>
<p>It comes bundled with a ton of helpful functions, helpers, plugins, themes, and a few things that make you shoutâ¦</p>
<blockquote>
<p>âOH MY ZSHELL!â</p>
</blockquote>
<h2>
<a id="user-content-setup" class="anchor" href="#setup" aria-hidden="true"><span class="octicon octicon-link"></span></a>Setup</h2>
<p><code>oh-my-zsh</code> should work with any recent release of <a href="http://www.zsh.org/">Zsh</a>. The minimum recommended version is <strong>4.3.9</strong>.</p>
<p>If not already installed, you can install <a href="http://www.zsh.org/">Zsh</a> using the command-line.</p>
<h3>
<a id="user-content-the-automatic-installer-do-you-trust-me" class="anchor" href="#the-automatic-installer-do-you-trust-me" aria-hidden="true"><span class="octicon octicon-link"></span></a>The automatic installerâ¦ do you trust me?</h3>
<p>You can install this via the command-line with either <code>curl</code> or <code>wget</code>.</p>
<h4>
<a id="user-content-via-curl" class="anchor" href="#via-curl" aria-hidden="true"><span class="octicon octicon-link"></span></a>via <code>curl</code>:</h4>
<p><code>curl -L http://install.ohmyz.sh | sh</code></p>
<h4>
<a id="user-content-via-wget" class="anchor" href="#via-wget" aria-hidden="true"><span class="octicon octicon-link"></span></a>via <code>wget</code>:</h4>
<p><code>wget --no-check-certificate http://install.ohmyz.sh -O - | sh</code></p>
<h4>
<a id="user-content-optionally-change-the-install-directory" class="anchor" href="#optionally-change-the-install-directory" aria-hidden="true"><span class="octicon octicon-link"></span></a>
<strong>Optionally</strong>, change the install directory:</h4>
<p>The default location is <code>~/.oh-my-zsh</code> (hidden in your home directory).</p>
<p>You can change the install directory with the <code>ZSH</code> environment variable, either by running <code>export ZSH=/your/path</code> before installing, or by setting it before the end of the install pipeline like this:</p>
<p><code>curl -L https://raw.github.com/robbyrussell/oh-my-zsh/master/tools/install.sh | ZSH=~/.dotfiles/zsh sh</code></p>
<h3>
<a id="user-content-the-manual-way" class="anchor" href="#the-manual-way" aria-hidden="true"><span class="octicon octicon-link"></span></a>The manual way</h3>
<p>1. Clone the repository:</p>
<p><code>git clone git://github.com/robbyrussell/oh-my-zsh.git ~/.oh-my-zsh</code></p>
<p>2. <strong>Optionally</strong>, backup your existing <code>~/.zshrc</code> file:</p>
<p><code>cp ~/.zshrc ~/.zshrc.orig</code></p>
<p>3. Create a new <a href="http://www.zsh.org/">Zsh</a> config file by copying the <a href="http://www.zsh.org/">Zsh</a> template weâve provided:</p>
<p><code>cp ~/.oh-my-zsh/templates/zshrc.zsh-template ~/.zshrc</code></p>
<p>4. Set <a href="http://www.zsh.org/">Zsh</a> as your default shell:</p>
<p><code>chsh -s /bin/zsh</code></p>
<p>5. Start or restart <a href="http://www.zsh.org/">Zsh</a> by opening a new command-line window.</p>
<h3>
<a id="user-content-problems" class="anchor" href="#problems" aria-hidden="true"><span class="octicon octicon-link"></span></a>Problems?</h3>
<p>You <em>might</em> need to modify your <code>PATH</code> in <code>~/.zshrc</code> if youâre not able to find some commands after switching to <code>oh-my-zsh</code>.</p>
<p>If you installed manually or changed the install location, check the <code>ZSH</code> environment variable in <code>~/.zshrc</code>.</p>
<h2>
<a id="user-content-usage" class="anchor" href="#usage" aria-hidden="true"><span class="octicon octicon-link"></span></a>Usage</h2>
<ul>
	<li>enable the plugins you want in your <code>~/.zshrc</code> (take a look at the <code>plugins/</code> directory and the <a href="https://github.com/robbyrussell/oh-my-zsh/wiki/Plugins">wiki</a> to see whatâs available)
	<ul>
		<li>example: <code>plugins=(git osx ruby)</code>

	</ul>
</li>
	<li>theme support: change the <code>ZSH_THEME</code> environment variable in <code>~/.zshrc</code>
	<ul>
		<li>take a look at the <code>themes/</code> directory and the <a href="https://wiki.github.com/robbyrussell/oh-my-zsh/themes">wiki</a> to see what comes bundled with <code>oh-my-zsh</code>
</li>
	</ul>
</li>
	<li>&amp; much, much moreâ¦ take a look at the <code>lib/</code> directory to see what <code>oh-my-zsh</code> has to offerâ¦</li>

<h3>
<a id="user-content-customization" class="anchor" href="#customization" aria-hidden="true"><span class="octicon octicon-link"></span></a>Customization</h3>
<p>If you want to override any of the default behaviors, just add a new file (ending in <code>.zsh</code>) in the <code>custom/</code> directory.</p>
<p>If you have many functions that go well together, you can put them as a <code>*.plugin.zsh</code> file in the <code>custom/plugins/</code> directory and then enable this plugin (see â<a href="https://github.com/robbyrussell/oh-my-zsh#usage">Usage</a>â above).</p>
<p>If you would like to override the functionality of a plugin distributed with <code>oh-my-zsh</code>, create a plugin of the same name in the <code>custom/plugins/</code> directory and it will be loaded instead of the one in <code>plugins/</code>.</p>
<h3>
<a id="user-content-updates" class="anchor" href="#updates" aria-hidden="true"><span class="octicon octicon-link"></span></a>Updates</h3>
<p>By default you will be prompted to check for upgrades. If you would like <code>oh-my-zsh</code> to automatically upgrade itself without prompting you, set the following in your <code>~/.zshrc</code>:</p>
<p><code>DISABLE_UPDATE_PROMPT=true</code></p>
<p>To disable upgrades entirely, set the following in your <code>~/.zshrc</code>:</p>
<p><code>DISABLE_AUTO_UPDATE=true</code></p>
<p>To upgrade directly from the command-line, just run <code>upgrade_oh_my_zsh</code>.</p>
<h3>
<a id="user-content-uninstalling" class="anchor" href="#uninstalling" aria-hidden="true"><span class="octicon octicon-link"></span></a>Uninstalling</h3>
<p>If you want to uninstall <code>oh-my-zsh</code>, just run <code>uninstall_oh_my_zsh</code> from the command-line and itâll remove itself and revert you to <code>bash</code> (or your previous <a href="http://www.zsh.org/">Zsh</a> configuration).</p>
